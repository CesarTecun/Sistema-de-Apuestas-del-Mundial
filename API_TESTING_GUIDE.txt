EJEMPLO COMPLETO - APP DE LIGAS:
================================

1. Crear app:
python manage.py startapp ligas

2. En ligas/models.py:
from django.db import models
from django.contrib.auth.models import User

class Liga(models.Model):
    nombre = models.CharField(max_length=100)
    administrador = models.ForeignKey(User, on_delete=models.CASCADE)
    tipo = models.CharField(max_length=20, choices=[('apuesta', 'Apuesta'), ('diversion', 'Diversión')])
    precio_participacion = models.DecimalField(max_digits=10, decimal_places=2, null=True, blank=True)
    fecha_creacion = models.DateTimeField(auto_now_add=True)

3. En ligas/admin.py:
from django.contrib import admin
from .models import Liga

@admin.register(Liga)
class LigaAdmin(admin.ModelAdmin):
    list_display = ['nombre', 'administrador', 'tipo', 'precio_participacion', 'fecha_creacion']
    list_filter = ['tipo', 'fecha_creacion']
    search_fields = ['nombre']

4. En ligas/serializers.py:
from rest_framework import serializers
from .models import Liga

class LigaSerializer(serializers.ModelSerializer):
    class Meta:
        model = Liga
        fields = '__all__'

5. En ligas/views.py:
from rest_framework import viewsets
from .models import Liga
from .serializers import LigaSerializer

class LigaViewSet(viewsets.ModelViewSet):
    queryset = Liga.objects.all()
    serializer_class = LigaSerializer

6. En ligas/urls.py:
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import LigaViewSet

router = DefaultRouter()
router.register(r'ligas', LigaViewSet)

urlpatterns = [
    path('', include(router.urls)),
]

7. En backend/urls.py agregar:
path('api/ligas/', include('ligas.urls')),

8. Probar:
- Admin: http://127.0.0.1:8000/admin/ (crear ligas)
- API: http://127.0.0.1:8000/api/ligas/ (ver JSON)
- curl: curl http://127.0.0.1:8000/api/ligas/

NOTAS IMPORTANTES:
==================
- NO necesitan frontend para desarrollar el backend
- Pueden probar TODO desde el Admin Panel
- Los endpoints REST son para cuando el frontend necesite consumir datos
- SQLite es temporal, cambiarán a PostgreSQL con Docker

COMUNICACIÓN CON EQUIPO FRONTEND:
=================================
Cuando terminen un módulo, comuniquen:
1. Endpoint URL (ej: /api/ligas/)
2. Método HTTP (GET, POST, PUT, DELETE)
3. Estructura JSON request/response
4. Autenticación requerida (si/no)

Ejemplo:
POST /api/ligas/
Request: {"nombre": "Liga Mundial", "tipo": "apuesta", "precio_participacion": 100}
Response: {"id": 1, "nombre": "Liga Mundial", ...}
Auth: Requiere login
